     7тЖТ     7тЖТ## Project Overview
     8тЖТ     8тЖТ
     9тЖТ     9тЖТCampusKart is a full-stack college marketplace web app. **Note on stack:**
    10тЖТ    10тЖТThe user requested Firebase, but the sandbox is preconfigured with
    11тЖТ    11тЖТPrisma + SQLite and has no real Firebase credentials. We therefore build a
    12тЖТ    12тЖТfully-functional equivalent on the available stack:
    13тЖТ    13тЖТ
    14тЖТ    14тЖТ- **Auth** тЖТ cookie-based session (`src/lib/session.ts`) with a *simulated*
    15тЖТ    15тЖТ  Google Sign-In modal (`src/components/site/auth-modal.tsx`). The
    16тЖТ    16тЖТ  `/api/auth/login` endpoint creates/finds a user by email and sets an
    17тЖТ    17тЖТ  httpOnly signed session cookie.
    18тЖТ    18тЖТ- **Database** тЖТ Prisma + SQLite (`prisma/schema.prisma`), mirroring the
    19тЖТ    19тЖТ  requested Firestore collections as tables (User, College, Product,
    20тЖТ    20тЖТ  WritingPost, Conversation, Message, ConversationRead, Report).
    21тЖТ    21тЖТ- **Storage** тЖТ local file uploads to `public/uploads/` via `/api/upload`,
    22тЖТ    22тЖТ  with client-side compression via `browser-image-compression`.
    23тЖТ    23тЖТ- **Real-time chat** тЖТ polling (every ~3s) instead of Firestore listeners.
    24тЖТ    24тЖТ
    25тЖТ    25тЖТAll functional requirements are met. A README + `.env.example` will be added
    26тЖТ    26тЖТdocumenting how to migrate to Firebase.

    28тЖТ    28тЖТ## Architecture / Conventions (READ THIS)
    29тЖТ    29тЖТ
    30тЖТ    30тЖТ- **Single user-visible route**: only `/` is shown to the user. Navigation is
    31тЖТ    31тЖТ  done via query params using the `useNav()` hook
    32тЖТ    32тЖТ  (`src/hooks/use-nav.ts`):
    33тЖТ    33тЖТ  - `?view=marketplace` (default)
    34тЖТ    34тЖТ  - `?view=product&id=<productId>`

    36тЖТ    36тЖТ  - `?view=chat&conv=<conversationId>`

    39тЖТ    39тЖТ  - Extra params allowed (e.g. `tab=listings`).
    40тЖТ    40тЖТ- **Theme**: green primary color already configured in `globals.css`.
    41тЖТ    41тЖТ  Use `bg-primary`, `text-primary`, etc. Do NOT use indigo/blue.
    42тЖТ    42тЖТ- **Sticky footer**: the root wrapper in `page.tsx` uses
    43тЖТ    43тЖТ  `min-h-screen flex flex-col`; the `<Footer>` has `mt-auto`.
    44тЖТ    44тЖТ- **Auth**: `useAuth()` zustand store (`src/store/auth-store.ts`) holds the

    46тЖТ    46тЖТ  When `user` is null тЖТ show `AuthModal`. When `user.onboarded === false` тЖТ
    47тЖТ    47тЖТ  show `OnboardingModal`.
    48тЖТ    48тЖТ- **Toasts**: use `const { toast } = useToast()` from `@/hooks/use-toast` for
    49тЖТ    49тЖТ  all user actions.
    50тЖТ    50тЖТ- **API helper**: there is no shared client fetch helper; use `fetch` directly.
    51тЖТ    51тЖТ  All API routes return JSON. Mutations return the updated entity.
    52тЖТ    52тЖТ- **Image URLs**: stored as JSON string in `Product.images` /
    53тЖТ    53тЖТ  `WritingPost` (n/a). Parse with `JSON.parse` on read. Empty тЖТ `[]`.

    55тЖТ    55тЖТ  `format(new Date(x), "d MMM yyyy")`.

   195тЖТ   195тЖТAPI routes created (all under `src/app/api/conversations/**`):
   196тЖТ   196тЖТ
   197тЖТ   197тЖТ1. `src/app/api/conversations/route.ts`
   198тЖТ   198тЖТ   - `GET /api/conversations` тАФ protected (401 if no session). Returns `{ conversations: Array<Conversation & { unread: number; otherName?: string }> }`. Query: `where: { participants: { contains: user.id } }` (substring match on the JSON-stringified participants works because cuid ids never appear as substrings of each other), ordered by `lastMessageAt desc`. For each conversation, computes `unread` = count of messages where `senderId != user`, `read = false`, AND `createdAt > user's ConversationRead.lastReadAt` (or all such unread if no read row). `otherName` is resolved by bulk-fetching the "other" participant's name from `User` (single `findMany` with `id IN (...)`, then map). `ConversationRead` rows are also bulk-fetched in one query to avoid N+1.
   199тЖТ   199тЖТ   - `POST /api/conversations` тАФ protected. Body `{ contextType, contextId, contextTitle, participantId }`. Validates `contextType тИИ {PRODUCT, WRITING}`, requires `contextId` and `participantId`, rejects self-conversations (400 "Cannot start a conversation with yourself.") and non-existent participants (404). Find-or-create: queries conversations by `contextId`, then filters in JS (SQLite can't do JSON array membership) for one whose parsed `participants` includes both `user.id` and `participantId`.

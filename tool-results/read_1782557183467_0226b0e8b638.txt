     8ŌåÆ
     9ŌåÆCampusKart is a full-stack college marketplace web app. **Note on stack:**
    10ŌåÆThe user requested Firebase, but the sandbox is preconfigured with
    11ŌåÆPrisma + SQLite and has no real Firebase credentials. We therefore build a
    12ŌåÆfully-functional equivalent on the available stack:
    13ŌåÆ
    14ŌåÆ- **Auth** ŌåÆ cookie-based session (`src/lib/session.ts`) with a *simulated*
    15ŌåÆ  Google Sign-In modal (`src/components/site/auth-modal.tsx`). The
    16ŌåÆ  `/api/auth/login` endpoint creates/finds a user by email and sets an
    17ŌåÆ  httpOnly signed session cookie.
    18ŌåÆ- **Database** ŌåÆ Prisma + SQLite (`prisma/schema.prisma`), mirroring the
    19ŌåÆ  requested Firestore collections as tables (User, College, Product,
    20ŌåÆ  WritingPost, Conversation, Message, ConversationRead, Report).
    21ŌåÆ- **Storage** ŌåÆ local file uploads to `public/uploads/` via `/api/upload`,
    22ŌåÆ  with client-side compression via `browser-image-compression`.
    23ŌåÆ- **Real-time chat** ŌåÆ polling (every ~3s) instead of Firestore listeners.
    24ŌåÆ
    25ŌåÆAll functional requirements are met. A README + `.env.example` will be added
    26ŌåÆdocumenting how to migrate to Firebase.

    28ŌåÆ## Architecture / Conventions (READ THIS)
    29ŌåÆ
    30ŌåÆ- **Single user-visible route**: only `/` is shown to the user. Navigation is
    31ŌåÆ  done via query params using the `useNav()` hook
    32ŌåÆ  (`src/hooks/use-nav.ts`):
    33ŌåÆ  - `?view=marketplace` (default)
    34ŌåÆ  - `?view=product&id=<productId>`
    35ŌåÆ  - `?view=writing`
    36ŌåÆ  - `?view=chat&conv=<conversationId>`
    37ŌåÆ  - `?view=profile`
    38ŌåÆ  - `?view=new-listing`, `?view=new-writing`
    39ŌåÆ  - Extra params allowed (e.g. `tab=listings`).
    40ŌåÆ- **Theme**: green primary color already configured in `globals.css`.
    41ŌåÆ  Use `bg-primary`, `text-primary`, etc. Do NOT use indigo/blue.
    42ŌåÆ- **Sticky footer**: the root wrapper in `page.tsx` uses
    43ŌåÆ  `min-h-screen flex flex-col`; the `<Footer>` has `mt-auto`.
    44ŌåÆ- **Auth**: `useAuth()` zustand store (`src/store/auth-store.ts`) holds the
    45ŌåÆ  current user. `user.loading` is true until first `/api/auth/me` resolves.
    46ŌåÆ  When `user` is null ŌåÆ show `AuthModal`. When `user.onboarded === false` ŌåÆ
    47ŌåÆ  show `OnboardingModal`.
    48ŌåÆ- **Toasts**: use `const { toast } = useToast()` from `@/hooks/use-toast` for
    49ŌåÆ  all user actions.
    50ŌåÆ- **API helper**: there is no shared client fetch helper; use `fetch` directly.
    51ŌåÆ  All API routes return JSON. Mutations return the updated entity.
    52ŌåÆ- **Image URLs**: stored as JSON string in `Product.images` /
    53ŌåÆ  `WritingPost` (n/a). Parse with `JSON.parse` on read. Empty ŌåÆ `[]`.
    54ŌåÆ- **Date formatting**: use `date-fns` (already installed). e.g.
    55ŌåÆ  `format(new Date(x), "d MMM yyyy")`.

   195ŌåÆAPI routes created (all under `src/app/api/conversations/**`):
   196ŌåÆ
   197ŌåÆ1. `src/app/api/conversations/route.ts`
   198ŌåÆ   - `GET /api/conversations` ŌĆö protected (401 if no session). Returns `{ conversations: Array<Conversation & { unread: number; otherName?: string }> }`. Query: `where: { participants: { contains: user.id } }` (substring match on the JSON-stringified participants works because cuid ids never appear as substrings of each other), ordered by `lastMessageAt desc`. For each conversation, computes `unread` = count of messages where `senderId != user`, `read = false`, AND `createdAt > user's ConversationRead.lastReadAt` (or all such unread if no read row). `otherName` is resolved by bulk-fetching the "other" participant's name from `User` (single `findMany` with `id IN (...)`, then map). `ConversationRead` rows are also bulk-fetched in one query to avoid N+1.
   199ŌåÆ   - `POST /api/conversations` ŌĆö protected. Body `{ contextType, contextId, contextTitle, participantId }`. Validates `contextType Ōłł {PRODUCT, WRITING}`, requires `contextId` and `participantId`, rejects self-conversations (400 "Cannot start a conversation with yourself.") and non-existent participants (404). Find-or-create: queries conversations by `contextId`, then filters in JS (SQLite can't do JSON array membership) for one whose parsed `participants` includes both `user.id` and `participantId`. Returns existing (200) or creates new with `participants: JSON.stringify([user.id, participantId])`, `lastMessage: ""`, `lastMessageAt: now`, `lastSenderId: null` (201). Idempotent ŌĆö verified end-to-end that a second POST with the same contextId+participantId returns the original conversation.
   200ŌåÆ
   201ŌåÆ2. `src/app/api/conversations/[id]/route.ts`
   202ŌåÆ   - `GET` ŌĆö protected. Verifies the current user is a participant (404 if not found OR not a participant ŌĆö never reveals existence to non-participants). Returns `{ conversation }` parsed, with `otherName` resolved from the `User` table.
   203ŌåÆ
   204ŌåÆ3. `src/app/api/conversations/[id]/messages/route.ts`
   205ŌåÆ   - `GET` ŌĆö protected; verify participant. Returns `{ messages: Message[] }` ordered by `createdAt asc`, capped at 200 for safety.
   206ŌåÆ   - `POST` ŌĆö protected; verify participant. Body `{ content }` (required, max 4000 chars). Creates the message (`senderId`/`senderName` from session, `read:false`). Updates the conversation: `lastMessage = content`, `lastMessageAt = now`, `lastSenderId = senderId`. Upserts `ConversationRead` for the sender with `lastReadAt = now` (so the sender's own unread badge stays 0). Also flips `read=true` on all messages in this conversation sent by the OTHER party that are older ŌĆö nice-to-have so the recipient's read flags flip when the sender replies. Returns `{ message }` (201). A small `authorize()` helper centralizes the participant check and is reused by both handlers.
   207ŌåÆ
   208ŌåÆ4. `src/app/api/conversations/[id]/read/route.ts`
   209ŌåÆ   - `POST` ŌĆö protected; verify participant. Runs a Prisma `$transaction` that (a) upserts `ConversationRead` for the current user with `lastReadAt = now`, and (b) `updateMany` flips `read=true` on all messages in this conversation where `senderId != currentUserId` and `read = false`. Returns `{ ok: true }`.
